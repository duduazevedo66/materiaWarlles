Capítulo 2

O CSS (Cascading Style Sheets), HTML (Hypertext Markup Language) e o JavaScript são tecnologias fundamentais de desenvolvimento front-end, onde são executadas diretamente no navegador do usuário.

2. Versões do HTML

Fazendo uma leve introdução sobre a Linguagem de Marcação de Hipertexto, o HTML é a linguagem de marcação padrão para documentos projetados para serem expostos em navegadores web. Geralmente também chega a ser auxiliada por tecnologias como CSS e também pelo JavaScript. O HTML pode incorporar linguagens de programação como o próprio JavaScript já citado logo acima, afetando o comportamento e até mesmo o conteúdo existente nas páginas da web. Já o CSS que também na grande maioria das vezes se faz presente, define o layout e a aparência da página.

2.1 Versões

As principais versões do HTML são:

  • HTML (1992): Foi a primeira versão proposta por Tim Berners-Lee desde 1982.
  • HTML 4.0 (1997): Versão responsável por um marco na web, separando a estrutura do layout (usando CSS) e introduzindo suporte a multimídia, scripts e tabelas.
  • XHTML 1.0: Nessa versão houve uma reformulação do HTML onde passaram a existir regras mais rígidas baseadas no XML.
  • HTML5 (2014): É a versão mais moderna, responsável por trazer novas funcionalidades e diversas APIs web.

2.1.1 Usando elementos e atributos

A estrutura básica do HTML gira em torno de atributos e elementos. Um elemento é formado por uma tag de abertura e uma tag de fechamento, um exemplo é o <h1> abre um título de primeiro nível e </h1> o fecha. O conteúdo fica entre as duas tags. Já os atributos ficam dentro da tag de abertura e servem para personalizar o elemento, sendo compostos por um nome e um valor separados por sinal de igual (ex.: href="url").

2.1.3 Salvando Páginas Web como Documentos HTML

Páginas HTML são, no fundo, simples arquivos de texto com extensão .html ou .htm. O processo é direto: onde se escreve o código em um editor, salva-se o arquivo e abre ele no navegador via menu "Abrir arquivo", arrastando o arquivo para a janela do browser ou com um duplo clique. O autor também menciona as Developer Tools do Chrome como um recurso valioso para inspecionar e entender o código de qualquer página.

2.2. Títulos, Parágrafos e Outras Formatações de Texto

O HTML oferece seis níveis de título, de <h1> (maior) a <h6> (menor). Parágrafos são definidos com <p> e são exibidos com espaçamento automático entre eles. Para formatação de texto, há elementos como <b> (negrito), <i> (itálico), <strong> e <em> (ênfase semântica), <sup> (sobrescrito) e <sub> (subscrito). O <br> insere uma quebra de linha. Para citações, existem <blockquote> (citações longas) e <q> (citações curtas), além de <cite>, <abbr> e <dfn> para referências, abreviações e definições.

2.2.1 Criando Listas

Existem três tipos de listas em HTML. Listas ordenadas (<ol>) exibem os itens numerados; listas não ordenadas (<ul>) usam marcadores visuais (bullets); e listas de definição (<dl>) são formadas por pares de termo (<dt>) e descrição (<dd>). Em todos os casos, os itens individuais são criados com <li>, exceto nas listas de definição. Também é possível criar listas aninhadas, colocando uma lista dentro de um item de outra, o navegador cuida automaticamente da indentação e da mudança de símbolo.

2.2.2 Definindo Links

Links são criados com o elemento <a> e o atributo href. O capítulo distingue três tipos: links externos (apontam para outros sites usando URL completa, podendo abrir em nova aba com target="_blank"), links relativos (apontam para páginas do mesmo site sem precisar do domínio, usando caminhos como "../pagina.html" para navegar entre diretórios) e links internos (saltam para uma seção da mesma página usando âncoras, onde o destino recebe um id e o link usa href="#id"). O autor destaca a importância de usar textos de link descritivos e significativos, evitando expressões genéricas como "clique aqui".

2.2.3 Incluindo Imagens

Imagens são inseridas com o elemento <img>, que é um elemento vazio pois não tem tag de fechamento separada e é controlado apenas por atributos. O atributo src define o caminho ou URL da imagem, enquanto alt fornece um texto alternativo exibido caso a imagem não carregue, essencial para acessibilidade e SEO. Desde o HTML5, é possível agrupar uma imagem com sua legenda usando <figure> e <figcaption>, tornando o conteúdo mais semântico.

2.2.4 Estruturando Dados em Tabelas

Tabelas são criadas com o elemento <table>. Cada linha é definida por <tr>, as células de dados por <td> e os cabeçalhos por <th>. As colunas não são declaradas explicitamente pois elas surgem da combinação de linhas e células. Para tabelas maiores, os elementos <thead>, <tbody> e <tfoot> organizam a estrutura semanticamente e facilitam a estilização com CSS. Para mesclar células, usa-se colspan (expansão horizontal, entre colunas) e rowspan (expansão vertical, entre linhas).

2.2.5 Definindo Formulários

Formulários são essenciais para capturar dados do usuário e enviá-los ao servidor. São criados com <form> e podem conter campos de texto (input type="text"), senhas (type="password"), e-mail (type="email", com validação automática), botões de rádio (type="radio"), checkboxes (type="checkbox"), listas suspensas (<select> + <option>), áreas de texto (<textarea>) e botão de envio (type="submit"). O elemento <label> associa rótulos aos campos, e <fieldset> + <legend> agrupam seções do formulário. O atributo action define a URL de envio, e method define o método HTTP (geralmente POST).

2.2.6 Informações Adicionais

O autor fecha a seção 2.2 com uma série de conceitos que aparecem com frequência na prática. Elementos de bloco (como <h1>, <p>, <ul>) sempre começam em nova linha, enquanto elementos inline (como <a>, <b>, <img>) ficam na mesma linha do conteúdo ao redor. O <div> e o <span> são agrupadores genéricos, o primeiro em bloco, o segundo inline que são usados para criar componentes customizados. Caracteres especiais como < e > precisam ser escapados como &lt; e &gt;. O id identifica um elemento de forma única, e o class agrupa elementos para estilização com CSS. Comentários são escritos entre <!-- e -->. O HTML5 também trouxe elementos estruturais semânticos como <header>, <footer>, <nav>, <article>, <aside> e <section>, além de suporte a metadados via <meta> e inclusão de CSS e JavaScript via <link>, <style> e <script>.

2.3 Resumo e Perspectiva

2.3.1 Pontos-Chave

Logo no final do capítulo, o autor consolida as ideias centrais: HTML é uma linguagem de marcação que define estrutura e semântica através de elementos com tags de abertura e fechamento; atributos passam informações adicionais aos elementos; alguns elementos são vazios e não aceitam conteúdo; arquivos HTML são texto puro; e os elementos mais importantes envolvem títulos, parágrafos, listas, links, imagens, tabelas e formulários.

Capítulo 3

Resumo do Capitulo 3 - Projetando Páginas Web com CSS

3.1 Introdução

O CSS (Cascading Style Sheets) é a linguagem responsável por definir como os elementos HTML de uma página web são visualmente apresentados no navegador. Enquanto o HTML cuida da estrutura e do conteúdo, o CSS cuida da aparência como as cores, fontes, espaçamentos, bordas, posicionamento e muito mais. Sem CSS, uma página web exibe apenas texto e elementos básicos sem nenhum estilo.

3.1.1 O Principio do CSS

Uma regra CSS é composta por duas partes essenciais: o seletor e a declaração. O seletor indica quais elementos HTML serão afetados pela regra. A declaração, escrita entre chaves, especifica como esses elementos devem ser exibidos, ela é formada por uma propriedade e um valor, separados por dois-pontos e finalizados com ponto e vírgula. Por exemplo, a regra h1 { color: darkblue; } aplica a cor azul escuro a todos os títulos de primeiro nível.

3.1.2 Incluindo CSS no HTML

Existem três formas de incluir CSS em um documento HTML:

  • CSS Externo: código em arquivo separado .css vinculado via <link>. Abordagem mais recomendada, pois permite reutilização e facilita manutenção.
  • CSS Interno: regras escritas diretamente no <head> dentro da tag <style>. Útil para estilizar uma única página.
  • CSS Inline: estilo definido no atributo style de um elemento. Deve ser evitado na maioria dos casos.

3.1.3 Seletores

Os seletores definem quais elementos HTML serão afetados:

  • Type selector: Seleciona pela tag (ex.: h1, input)
  • Class selector: Seleciona por classe (ex.: .destaque)
  • ID selector: Seleciona um elemento único por id (ex.: #menu)
  • Universal selector: Seleciona todos os elementos (*)
  • Attribute selector: Seleciona por valor de atributo (ex.: [type="text"])
  • Sibling selectors: Seleciona irmãos (h1 + p ou h1 ~ p)
  • Child / Descendant: Seleciona filhos diretos (body > ul) ou qualquer descendente (body ul)

3.1.4 Cascata e Especificidade

Quando duas ou mais regras se aplicam ao mesmo elemento, prevalece a regra com maior especificidade. A especificidade é calculada por: IDs valem mais que classes e atributos, que valem mais que tipos de elementos. Estilos inline têm a maior especificidade.

3.1.5 Herança

Propriedades como color e font-family são herdadas pelos elementos filhos automaticamente. Já propriedades como border não são herdadas.

3.2 Aplicando Cores e Formatação de Texto

Cores: color (texto) e background-color (fundo).
Formatos: nome, hexadecimal, RGB, RGBA, HSL, HSLA.

Formatação de texto: font-family, font-size, font-style, font-weight, text-transform, text-decoration, line-height, letter-spacing, word-spacing, text-align.

3.3 Listas e Tabelas

Listas: list-style-type (disco, círculo, quadrado, números, letras, romanos), list-style-image (imagem customizada), list-style-position (outside/inside).

Tabelas: border, border-collapse: collapse, padding, pseudo-classes como :nth-child(odd) e :nth-child(even) para efeito zebrado.

3.4 Compreendendo os Diferentes Sistemas de Layout

Posicionamento: Elementos block-level (ex.: <div>) e inline-level (ex.: <span>). Propriedade position com valores: static, relative, absolute, fixed, sticky, inherit.

Flexbox: Sistema unidimensional. Ativado com display: flex. Propriedades: flex-direction, justify-content, align-items, flex-grow, flex-shrink, flex-basis, order.

Grid Layout: Sistema bidimensional (linhas e colunas). Ativado com display: grid. Propriedades: grid-template-columns, grid-template-rows, grid-column, grid-row, gap.

3.5 Resumo e Perspectivas

Pontos-chave: regra CSS (seletor + declaração), formas de inclusão, seletores, cascata e especificidade, herança, formatação de textos, listas, tabelas, e sistemas de layout (Float, Flexbox e Grid).
